# Сервис аренды СИМ — симуляция устройств + геозоны

Учебный (дипломный) проект: бэкенд сервиса кикшеринга с акцентом на две
«глубокие» технические фичи:

1. **IoT-симуляция через MQTT** — устройства эмулируются отдельным процессом и
   общаются с бэкендом ровно так, как это делало бы реальное железо.
2. **Геозоны (geofencing) на PostGIS** — пространственные запросы определяют,
   где можно ездить, где действует ограничение скорости и где парковка запрещена.

## Архитектура

```
┌────────────────┐   telemetry    ┌──────────────────┐   SQL/PostGIS   ┌────────────┐
│  Эмулятор СИМ   │ ─────────────▶ │   MQTT-брокер     │                │ PostgreSQL │
│ (emulator/)     │ ◀───────────── │  (Mosquitto)      │                │  + PostGIS │
└────────────────┘    commands    └────────┬─────────┘                 └─────▲──────┘
                                            │ telemetry / commands             │
                                   ┌────────▼──────────┐  geofencing            │
                                   │  FastAPI-бэкенд    │ ───────────────────────┘
                                   │  (app/)            │
                                   │  + WebSocket карта │
                                   └────────────────────┘
```

Поток одной телеметрии: эмулятор публикует координаты → бэкенд получает их из
MQTT → прогоняет точку через геозоны (PostGIS) → обновляет устройство в БД →
при необходимости шлёт команду обратно (`lock`, `set_speed_limit`) → транслирует
позицию в WebSocket для live-карты.

## Структура

| Путь | Назначение |
|------|------------|
| `app/models.py` | ORM-модели, в т.ч. гео-колонки (Point у устройства, Polygon у зоны) |
| `app/geofencing.py` | вся пространственная логика (ST_Contains, ST_DWithin, ST_Distance) |
| `app/mqtt_client.py` | MQTT-мост: приём телеметрии, проверка геозон, отправка команд |
| `app/main.py` | FastAPI: REST-эндпоинты + WebSocket live-карты |
| `app/seed.py` | наполнение БД тестовыми устройствами и тремя зонами |
| `emulator/emulator.py` | эмулятор устройств (заменяет реальное железо) |
| `app/admin.py` | API оператора: список парка и команды (lock/unlock/ремонт) |
| `app/static/admin.html` | веб-панель оператора: живая карта устройств + геозоны |

> Запуск под **Windows** (Docker Desktop или нативно) описан в `README_WINDOWS.md`.

## Геозоны

- **slow** — медленная зона; задаёт `speed_limit` (км/ч). Итоговый лимит в точке =
  минимум по всем накрывающим её медленным зонам, иначе глобальный дефолт.
- **forbidden** — запрещённая зона; въезд во время поездки → авто-команда `lock`,
  завершить поездку внутри нельзя (HTTP 422).
- **parking** — разрешённая парковка (для будущей логики «верни в зону парковки»).

## MQTT-топики

| Топик | Направление | Содержимое |
|-------|-------------|------------|
| `devices/{code}/telemetry` | устройство → бэкенд | `{lat, lng, battery, speed}` |
| `devices/{code}/commands`  | бэкенд → устройство | `{command: unlock\|lock\|set_speed_limit, ...}` |

## REST API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/devices/nearby?lat&lng&radius` | свободные устройства в радиусе (метры) |
| GET | `/zones` | все геозоны как GeoJSON FeatureCollection |
| GET | `/devices/{code}/geofence` | статус геозон для устройства сейчас |
| POST | `/trips/start?user_id&device_code` | начать поездку (unlock) |
| POST | `/trips/{id}/end` | завершить (с проверкой запрещённой зоны) |
| WS | `/ws/devices` | live-поток позиций |

Документация Swagger доступна на `http://localhost:8000/docs`
## Запуск

```bash
docker compose up --build        # поднимет db, mqtt, api, emulator
# в отдельном терминале — наполнить БД (один раз):
docker compose exec api python -m app.seed
```

Затем:
- открыть **панель оператора**: `http://localhost:8000/admin` (токен `admin-secret`);
- открыть `http://localhost:8000/docs`;
- `GET /zones` — увидеть три зоны;
- `GET /devices/nearby?lat=51.3397&lng=12.3731&radius=800` — найти устройства;
- `POST /trips/start` — разблокировать устройство (эмулятор начнёт «ехать»);
- наблюдать в логах эмулятора смену лимита скорости и блокировку при въезде
  в запрещённую зону.

## Что можно подчеркнуть на защите

- Реальный паттерн IoT-связи (телеметрия/команды через брокер), а не заглушки.
- Корректные геозапросы на PostGIS с GiST-индексами и расчётом расстояний в метрах.
- Асинхронная обработка (FastAPI + asyncio) и live-обновление карты через WebSocket.

## Возможные расширения (если будет время)

- Хранить маршрут поездки как `LineString` и считать дистанцию через `ST_Length`.
- Динамические зоны (например, временное ограничение скорости на время мероприятия).
- Простой фронт на Leaflet: отрисовать `/zones` полигонами и обновлять маркеры по WebSocket.
